## Summary

Adds the freemium model: **Free** (default, 15 probes/month) and **Pro** ($29/mo, unlimited + priority speed + exports + full per-agent detail). Stripe Checkout (hosted) for payments, Stripe Customer Portal for self-service cancel.

| Plan | Probes/mo | Probe speed | Per-agent detail | Exports | History |
|---|---|---|---|---|---|
| Free | 15 | 60-90s (3 waves) | hidden | 🔒 | 7 days |
| **Pro** | **Unlimited** | **~15s (1 wave)** ← original PRD target | full | PDF/CSV/JSON | unlimited |

## What changed

### Backend

| `backend/main.py` | `/api/probe` now reads user, enforces 402-with-detail when free quota is exceeded, increments counter on success, and threads `tier` into `run_full_probe`. `run_full_probe(tier="pro")` runs all 24 agents in one concurrent wave (no stagger). |
| `backend/report_export.py` | Exports (`/json`, `/csv`, `/pdf`) gated to Pro via `_require_pro` dep — returns 401 for anon, 402 for free. |
| `backend/supabase_client.py` | `save_probe(session_data, user_id=None)` — optional `user_id` argument, falls back if column missing. |

## One-time setup before merging

### Stripe Dashboard (5 min)

1. The Pro Product + Price (`price_1TbQtuEX9Slvr2kwpbhWvTlk`) already exists in test mode — no action needed there.
2. **Developers → Webhooks → Add endpoint**:
   - Endpoint URL: `<your-backend-public-url>/api/billing/webhook` (e.g. once you deploy backend to Render: `https://jacobi-backend.onrender.com/api/billing/webhook`)
   - Events to listen for: `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`
   - Click "Add endpoint", then copy the **Signing secret** (starts with `whsec_`)
3. **Settings → Billing → Customer portal**: Activate the portal. Default settings are fine; lets users cancel themselves.

### Supabase Dashboard

1. **SQL Editor**: paste the contents of `supabase/migrations/202605262030_add_user_id_and_billing_columns.sql` and run.
2. **Project Settings → API**: copy the **`service_role` secret** (not the anon key) — backend only.

### Env values to set

**Backend** (`Jacobi/.env.local` or `Jacobi/backend/.env`):
```
STRIPE_SECRET_KEY=sk_test_…
STRIPE_PRO_PRICE_ID=price_1TbQtuEX9Slvr2kwpbhWvTlk
STRIPE_WEBHOOK_SECRET=whsec_…           ← from step 2 above
SUPABASE_URL=https://dlxfhoquysrncxkelyxa.supabase.co
SUPABASE_SERVICE_KEY=eyJ…               ← from Supabase API settings
FRONTEND_URL=http://localhost:3000      ← change to deployed Vercel URL in prod
FREE_MONTHLY_PROBES=15
```

**Frontend** (`Jacobi/frontend/.env.local`):
```
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_…
```
(Same `NEXT_PUBLIC_SUPABASE_*` as before.)

### Local webhook testing

For testing the webhook locally without deploying the backend, install the Stripe CLI and run:
```
stripe listen --forward-to http://localhost:8000/api/billing/webhook
```
It prints a temporary `whsec_*` to use as `STRIPE_WEBHOOK_SECRET`.
