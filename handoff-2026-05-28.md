## Current State

JACOBI is a 24-agent adversarial pricing topology probe. It detects hidden pricing discrimination by deploying 24 parallel agents with unique digital fingerprints (location, device, cookies, referrer, network tier) against any pricing page, then uses statistical analysis + multi-provider AI to determine if the price varies based on identity.

## Features Implemented

### Core Pipeline
- 24-agent parallel probe in 3 staggered waves (datacenter → residential → mobile)
- Price parsing: JSON-LD → site-specific selectors → regex fallback
- Gradient computation: Welch's t-test with effect-size thresholds
- Topology classification: uniform / selective / progressive / aggressive
- AI provider cascade: AI/ML API (GPT-4o) → Gemini → OpenCode (DeepSeek) → Groq → heuristic fallback
- Deterministic savings verdict computation
- Cross-session memory via Cognee knowledge graph
- Fire-and-forget partner integrations (Cognee, TriggerWare)

### Frontend
- Chat/probe interface with real-time agent grid (24 cells, wave animation, per-cell status)
- ResultCard with collapsible sections (agents, histogram, comparison)
- Comparison view table (variable × high/low state × delta × significance)
- Topology badge export (SVG nutrition label, one-click copy)
- Shareable result URLs with dedicated `/share/[id]` page
- ErrorBoundary wrapping chat page
- About page, Pricing page, Leaderboard page, History page
- Reusable JacobiLogo, MatricesCursor, DotMatrix components

### Chrome Extension
- Manifest V3, right-click "Probe this price with JACOBI" (page + link)
- Popup with current URL, recent probes list, clear button
- Keyboard shortcuts: Ctrl+Shift+P (popup), Ctrl+Shift+L (quick probe)
- Floating price badge on product pages (shadow DOM, auto-detection)
- Desktop notifications on probe completion
- Daily probe counter badge on extension icon
- Settings page with backend URL config, clear data
- SVG icons (16/48/128) with [J] bracket logo

### Auth & Persistence
- Supabase Auth wired (Google OAuth + Email OTP)
- Dev sign-in via credentials
- Supabase persistence layer for probe results
- In-memory SESSION_STORE with TTL cleanup (30min expiry, 100 cap)
- Cognee knowledge graph for cross-session memory
- TriggerWare webhook dispatch for workflow automation

### Backend Infrastructure
- Rate limiting (5 req/min/IP on POST /api/probe)
- SESSION_STORE cleanup background task (every 5 min)
- Recurring probe scheduler background task (checks every 60s)
- _analysis_cache with TTL (1hr) + max size (100) + LRU eviction
- Non-US proxy retry logic (timeout → datacenter fallback)
- Site-specific price parsers (Google Flights, generic e-commerce)
- Parser registry for extensibility
- asyncio.to_thread() for sync AI calls from async handlers
- CORS enabled for all origins

## Key Architectural Decisions

1. **No agent framework** — After extensive debate (Metis + Momus), the pipeline architecture is correct for a deterministic 5-step flow. Agent framework would add latency, cost, and hallucination risk without architectural benefit.

2. **Fire-and-forget partners** — All partner integrations (Cognee, TriggerWare) are fire-and-forget with try/except. No partner can block the probe pipeline.

3. **Provider cascade** — AI providers chain from most capable to most available. Each failure falls through to the next. Statistical fallback always works.

4. **In-memory state** — SESSION_STORE, SCHEDULES, _analysis_cache, _rate_store are all in-memory. Simple and fast for hackathon scale. Session cleanup + TTLs prevent unbounded growth.
